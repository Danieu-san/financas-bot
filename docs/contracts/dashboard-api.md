# Dashboard API Contracts

All dashboard API routes are read-only and require `token` as a query parameter. The token is generated by the WhatsApp bot and carries the user id (`uid`); endpoints must never accept a user id from the client.

Invalid or expired tokens return:

```json
{ "error": "Token inválido ou expirado." }
```

## `GET /dashboard/api/kpis`

Returns:

```json
{
  "period": { "month": 4, "year": 2026 },
  "kpis": {
    "entradas": 1000,
    "saidas": 300,
    "cartoes": 100,
    "saldo": 600,
    "debtActiveCount": 1,
    "debtTotal": 1200
  },
  "topCategories": [{ "category": "Alimentação", "value": 300 }],
  "source": "sqlite"
}
```

## `GET /dashboard/api/cashflow`

Returns:

```json
{
  "period": { "month": 4, "year": 2026 },
  "dailyFlow": [{ "date": "10/05/2026", "entradas": 1000, "saidas": 400, "saldo": 600 }],
  "source": "sqlite"
}
```

Returns:

```json
{
  "alerts": [{ "level": "low", "code": "OK", "message": "Sem alerta crítico." }],
  "source": "sqlite"
```

## `GET /dashboard/api/summary`

Returns the complete user-scoped dashboard snapshot: `period`, `kpis`, `topCategories`, `dailyFlow`, `recentTransactions`, `goals`, `debts`, `alerts`, and `sync`.

## Safety Rules

- Token payload decides the user scope.
- Query parameters may choose period only (`month`, `year`), never user.
- No endpoint should log financial row contents.
- Prefer `source: "sqlite"`; `source: "memory"` is a fallback only.
